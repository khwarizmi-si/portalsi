# Contract gaps: Flutter dan Laravel

Dokumen ini mencatat perbedaan yang sudah diverifikasi dari source code, bukan asumsi desain. Laravel di `apps/api` tetap menjadi kontrak utama. Status diperbarui saat integrasi menemukan bukti baru.

## Ringkasan keputusan

## Usulan backend terpisah (tidak diterapkan dalam frontend)

1. Lindungi `/register-teachers`, `/register-parent`, dan `/fcm/register` dengan policy/middleware yang sesuai.
2. Tambahkan policy teacher/dev pada create announcement dan object policy portfolio.
3. Tambahkan cursor pagination untuk DM dan group messages.
4. Perbaiki filter privacy pada feed/explore di query server.
5. Tambahkan re-auth untuk delete account dan endpoint revoke session eksplisit.
6. Konsolidasikan channel/event realtime dan hapus event placeholder.
7. Tambahkan endpoint unregister FCM bila push web akan didukung.
8. Sediakan BFF/OAuth callback resmi untuk Akun RG tanpa client secret di browser.

## Aturan fallback UI

- Fitur tanpa kontrak server tidak diberi affordance yang terlihat tersinkron.
- Error third-party tidak menjadi error global aplikasi.
- `403 verified.api` menghasilkan CTA verifikasi email; `403` policy/ownership menghasilkan pesan izin, bukan CTA verifikasi.
- Empty search `404` dinormalisasi menjadi empty state hanya pada endpoint search yang memang memiliki perilaku tersebut.
- Untuk response shape yang berbeda, adapter domain khusus lebih dipilih daripada schema longgar atau `any`.
